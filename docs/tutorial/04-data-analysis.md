# Уровень 4 (бонус) — Анализ данных с embeddings

> Pipeline обработки векторных данных: очистка → нормализация → снижение размерности → кластеризация. Классика статистики применительно к современным embeddings.

## Полный Pipeline

```
Raw data → Embeddings → Normalization → PCA/UMAP → Clustering
(тексты)   (векторы)   (масштаб)        (снижение)  (группы)
```

---

## Этап 1 — Очистка данных

Мусор на входе = мусор на выходе.

| Что чистить | Как | Зачем |
|-------------|-----|-------|
| HTML-теги | `cheerio` / regex | Модель не обучалась на разметке |
| Дубликаты | `SELECT DISTINCT` / Set | Не раздуваем базу |
| Опечатки | Spell-checker | Опционально |
| Спец-символы, эмодзи | Regex | Если не несут смысла |
| Слишком короткие (<10 символов) | Фильтр | "ок", "да" — шум |
| Слишком длинные | Truncate/chunking | Модель не переварит |
| Разные языки в поле | Language detection | Отдельно обрабатывать |

⚠️ **Для embeddings НЕ нужно:**
- Stemming / lemmatization
- Удаление стоп-слов
- Токенизация

Модель всё это учла в обучении. Чисти только мусор, текст оставляй как есть.

## Этап 3 — Нормализация

### L2 нормализация векторов

$$v_{norm} = \frac{v}{\|v\|}$$

Делим вектор на его длину → единичный вектор.

**После L2:**
- Все векторы длины 1 (на единичной сфере)
- Косинусное сходство = скалярное произведение (быстрее)
- Евклидово расстояние = √(2 - 2·cos(θ))

```python
v_norm = v / np.linalg.norm(v)
```

### Нужно ли нормализовать embeddings?

**Уже нормализованы:** OpenAI, Cohere, bge-m3 отдают векторы с `||v|| ≈ 1`.

```python
np.linalg.norm(embedding)  # ≈ 1.0 для OpenAI
```

### Нормализация признаков (для смешанных векторов)

Если собираешь свой вектор из embedding + числовых признаков:

| Метод | Формула | Когда |
|-------|---------|-------|
| **Min-Max** | (x - min)/(max - min) | В диапазон [0, 1] |
| **Z-score** | (x - μ)/σ | Нормальное распределение |
| **Robust (IQR)** | (x - median)/IQR | Есть выбросы |
| **Log-transform** | log(x + 1) | Скошенные распределения (цены, просмотры) |

**Пример смешанного вектора для отелей:**
```
Vector = [
  embedding(description)[1536],    ← уже норм
  z_score(price),                   ← [500-500000₽] → [-2, +3]
  minmax(rating),                   ← [0-10] → [0, 1]
  scaled(latitude, longitude)       ← координаты
]
```

Без масштабирования числовые признаки задавят смысл текста или потеряются.

## Этап 4 — Снижение размерности

### Зачем

- 🚀 Скорость поиска (1536D → 128D = в 12 раз быстрее)
- 💾 Экономия памяти (миллион записей — гигабайты)
- 📊 Визуализация (2D/3D)
- 🎯 Убираем шум в малых компонентах
- 🧮 Кластеризация работает лучше (curse of dimensionality в 1536D)

### PCA — классика

```
1. Центрирование: X' = X - mean(X)
2. Ковариационная матрица: C = X'ᵀ · X' / (N-1)
3. Собственное разложение: C = V · Λ · Vᵀ
4. Берём K первых собственных векторов (макс. λ)
5. Проекция: X_reduced = X' · V[:, :K]
```

### Сколько компонент оставить

```python
from sklearn.decomposition import PCA

pca = PCA()
pca.fit(embeddings)

cumulative = np.cumsum(pca.explained_variance_ratio_)
k = np.argmax(cumulative >= 0.95) + 1
print(f"{k} компонент для 95% дисперсии")
```

**Эмпирика:**
- 95% дисперсии → стандарт
- 99% → максимально бережно
- 70% → агрессивно для визуализации

Для OpenAI 1536D обычно **128-256 компонент = 95% дисперсии**.

### Matryoshka (современная альтернатива)

OpenAI `text-embedding-3-*` обучены с MRL — просто отрезаешь первые N:

```python
full  = embedding[:1536]  # полный
mid   = embedding[:512]   # сжатый, почти без потерь
small = embedding[:256]   # ещё компактнее
```

Не требует PCA-обучения на своих данных.

### Альтернативы PCA

| Метод | Тип | Когда |
|-------|-----|-------|
| **PCA** | Линейный | Быстро, большинство задач |
| **UMAP** | 🔥 Нелинейный | Pre-processing для кластеризации, визуализация |
| **t-SNE** | Нелинейный | Только визуализация 2D/3D |
| **Autoencoder** | Глубокий | Редко, когда PCA не хватает |

**Для кластеризации лучше UMAP чем PCA** — сохраняет и локальную, и глобальную структуру.

## Этап 5 — Кластеризация

### K-means

```python
from sklearn.cluster import KMeans
kmeans = KMeans(n_clusters=10, random_state=42)
labels = kmeans.fit_predict(X_reduced)
```

- ✅ Быстро
- ❌ Нужно знать K
- ❌ Плохо с кластерами разной плотности

### HDBSCAN 🔥

```python
import hdbscan
clusterer = hdbscan.HDBSCAN(min_cluster_size=30)
labels = clusterer.fit_predict(X_reduced)
# -1 = шум, 0+ = кластеры
```

- ✅ K определяется сам
- ✅ Кластеры разной плотности
- ✅ Выбросы → -1
- ❌ Медленнее K-means

### Выбор K для K-means

**Метод локтя:**
```python
inertias = []
for k in range(2, 20):
    km = KMeans(n_clusters=k).fit(X)
    inertias.append(km.inertia_)
plt.plot(range(2, 20), inertias)  # ищем "локоть"
```

**Silhouette score:**
```python
from sklearn.metrics import silhouette_score
for k in range(2, 20):
    km = KMeans(n_clusters=k).fit(X)
    print(k, silhouette_score(X, km.labels_))
# Выше = лучше, диапазон [-1, 1]
```

### Другие методы

| Метод | Когда |
|-------|-------|
| **Agglomerative (hierarchical)** | Нужна иерархия тем |
| **Spectral Clustering** | Нелинейные границы |
| **DBSCAN** | Предшественник HDBSCAN, нужны параметры eps/minPts |
| **GMM** | Гауссовские кластеры (мягкая принадлежность) |

## Меры расстояния

| Метрика | Формула | Когда | pgvector |
|---------|---------|-------|----------|
| **Косинусная** | 1 - (A·B)/(\|\|A\|\|·\|\|B\|\|) | Embeddings | `<=>` |
| **Евклидова (L2)** | √Σ(Aᵢ-Bᵢ)² | Абсолютные значения важны | `<->` |
| **Манхэттенская (L1)** | Σ\|Aᵢ-Bᵢ\| | Устойчивость к выбросам | `l1_distance()` |
| **Скалярное произведение** | A·B | Нормализованные векторы | `<#>` |
| **Махаланобиса** | Ковариационная матрица | Учёт корреляций признаков | — |

Для embeddings почти всегда **косинусная**.

### Косинус vs Корреляция Пирсона

- **Косинус:** угол между векторами
- **Корреляция Пирсона:** косинус между **центрированными** векторами

Равны только для центрированных. Embeddings нормализованы по длине, но **не центрированы** → косинус ≠ корреляция Пирсона строго, но связь близкая.

## Практический пример — анализ жалоб клиентов

```python
import numpy as np
from sklearn.decomposition import PCA
import hdbscan
from openai import OpenAI

# 1. Очистка
def clean(text):
    text = re.sub(r'<[^>]+>', '', text)
    text = re.sub(r'\s+', ' ', text).strip()
    return text if len(text) > 20 else None

reviews = [r for r in (clean(raw) for raw in raw_reviews) if r]

# 2. Embeddings batch
client = OpenAI()
embeddings = []
for batch in chunked(reviews, 100):
    resp = client.embeddings.create(
        model="text-embedding-3-small",
        input=batch
    )
    embeddings.extend([d.embedding for d in resp.data])
embeddings = np.array(embeddings)

# 3. Нормализация (на всякий)
embeddings = embeddings / np.linalg.norm(embeddings, axis=1, keepdims=True)

# 4. Снижение размерности
pca = PCA(n_components=50, random_state=42)
X_reduced = pca.fit_transform(embeddings)
print(f"Дисперсия: {pca.explained_variance_ratio_.sum():.2%}")

# 5. Кластеризация
clusterer = hdbscan.HDBSCAN(min_cluster_size=30)
labels = clusterer.fit_predict(X_reduced)

# 6. Анализ
for cluster_id in set(labels):
    if cluster_id == -1:
        continue
    indices = np.where(labels == cluster_id)[0]
    print(f"Кластер {cluster_id} ({len(indices)} отзывов):")
    print(f"  Пример: {reviews[indices[0]]}")
    # LLM для автоматического названия
```

**Вывод:**
```
Кластер 0 (1200): "долгая доставка"
Кластер 1 (800):  "проблемы с оплатой"
Кластер 2 (450):  "брак товара"
...
Шум (350): не попали в кластеры
```

## Типовые задачи и pipeline

| Задача | Pipeline |
|--------|----------|
| **RAG-поиск** | Embeddings → pgvector + HNSW (без PCA/clustering) |
| **Рекомендации** | Embeddings → ближайшие по косинусу |
| **Сегментация клиентов/товаров** | Embeddings → UMAP → HDBSCAN → метки |
| **Анализ тем в текстах** | Embeddings → PCA/UMAP → HDBSCAN → LLM название кластера |
| **Визуализация данных** | Embeddings → UMAP 2D → scatter plot |
| **Быстрый поиск в огромной базе** | Embeddings → PCA 128D → IVFFlat/HNSW |
| **Детекция аномалий** | Embeddings → HDBSCAN → шум (-1) = аномалии |
| **Дедупликация** | Embeddings → cosine distance < threshold |

## Применение для Marpla (e-commerce)

Идеи использования embeddings+кластеризации в каталоге товаров селлеров:

1. **Сегментация товаров** — по описанию сгруппировать в категории автоматически
2. **Дедупликация** — найти одинаковые товары с разными названиями
3. **Рекомендации "похожие товары"** — быстрее чем через категории
4. **Детекция некачественных описаний** — короткие/пустые → аномалии
5. **Анализ конкурентов** — группировка по стилю описаний, ценовым нишам
6. **Автоматическое тегирование** — товар → ближайшие теги по смыслу
7. **SEO-кластеризация** — группы товаров с похожим SEO-стилем

## ⚠️ Подводные камни

| Проблема | Решение |
|----------|---------|
| Curse of dimensionality | Снижение размерности (PCA/UMAP) перед кластеризацией |
| Разные масштабы признаков | Нормализация (z-score / min-max) |
| K неизвестно заранее | HDBSCAN вместо K-means |
| Выбросы ломают кластеры | HDBSCAN помечает как шум (-1) |
| PCA теряет нелинейные зависимости | UMAP или autoencoder |
| Кластеры разной плотности | HDBSCAN вместо K-means/DBSCAN |
| Долгая индексация миллионов | Batch-обработка, локальные embeddings |

## Связь с классической статистикой

| Классика (2006+) | Embeddings (2026) |
|------------------|-------------------|
| Интервальные данные | Embeddings (непрерывные векторы) |
| Ручная кластеризация | HDBSCAN, UMAP + clustering |
| Линейные/нелинейные зависимости | Transformer-модель учит сама |
| Компонентный анализ (PCA) | PCA + Matryoshka (MRL) |
| Меры расстояния | Косинус, L2, L1, Mahalanobis |
| Иерархическая классификация | Hierarchical clustering на embeddings |

Методология та же — классификация, кластеризация, проекция. Меняется источник признаков (нейросеть вместо ручных) и масштаб (миллионы вместо сотен).
